# Project Health Check (February 2025)

این گزارش یک مرور سریع بر وضعیت فعلی پروژه است تا مطمئن شویم همه مؤلفه‌ها آماده ارائه و استفاده هستند.

## وضعیت فعلی
- بک‌اند Rust با فرمان `cargo test --locked` اجرا شد. ۵ تست واحد اجرا شد (۴ پاس، ۱ نادیده گرفته شده). هیچ تستی شکست نخورد و زمان اجرای بخش تست ۰.۰۲ ثانیه بود. 【85d1e4†L1-L15】
- هنگام بیلد هشدارهای متعددی وجود دارد (Import/Variable بدون استفاده و استفاده از `NaiveDateTime::timestamp`). برای پاک‌سازی بیلد، لازم است هشدارها برطرف شوند.
- بسته `sqlx-postgres` گزارش «future incompatibility» داده است؛ توصیه می‌شود گزارش کامل با `cargo report future-incompatibilities --id 2` بررسی شود تا در نسخه‌های آینده Rust مشکلی ایجاد نشود. 【85d1e4†L1-L5】

## پیش‌نیازهای کلیدی برای استفاده کامل
- فایل‌های `.env` باید در ریشه، `client-app` و `admin-app` قرار بگیرند (مقادیر نمونه در README آمده است).
- برای اجرای کلاینت‌ها به ابزارهای Tauri/Node و دسترسی به API/دیتابیس/Redis/MinIO نیاز است. قبل از اجرا از فعال بودن سرویس‌های بک‌اند اطمینان حاصل کنید.

## توصیه‌های بعدی
- هشدارهای بیلد را پاک‌سازی کنید (حذف importهای اضافه و جایگزینی روش timestamp در `auth.rs`).
- در صورت نیاز، تست‌های یکپارچه و E2E برای کلاینت‌ها اجرا یا اضافه شود تا پوشش قابلیت‌ها کامل شود.
- گزارش future-incompatibility اجرا و در صورت نیاز وابستگی‌ها به‌روزرسانی شود.
